# 深度研究 Skill — 横纵分析法

IRON LAW: 先摆事实，再给判断。每一个观点都必须有事实支撑；推测必须明确标注。

## 触发方式

当用户说以下内容时，激活此 skill：
- "深度研究XXX"
- "用横纵分析法研究XXX"
- "生成一份研究报告"
- "分析一下这个产品/公司/概念"
- "帮我做竞品分析"
- "出一份深度报告"
- "研究一下XXX的市场格局"

## 工作流程

### ⚠️ Step 1: 确认研究对象
确认用户要研究的是什么（产品/公司/技术概念/人物）。

### ⚠️ Step 2: 判断复杂度
根据研究对象的类型和背景，预估篇幅和重点：

| 类型 | 纵向重点 | 横向重点 | 预估篇幅 |
|------|---------|---------|---------|
| 产品/工具 | 版本迭代、用户增长、商业化 | 竞品功能对比、用户体验差异 | 10000-20000字 |
| 公司/组织 | 融资历程、战略转型、里程碑 | 市场竞争格局、差异化定位 | 15000-25000字 |
| 技术概念 | 提出背景、技术演进、标准之争 | 方案对比、优劣分析、趋势 | 10000-15000字 |
| 人物 | 职业轨迹、关键决策、成长脉络 | 同期人物对比、影响力评估 | 8000-15000字 |

### ⚠️ Step 3: 纵向分析（时间深度）

沿时间轴还原发展全貌，包含以下维度：

1. **起源追溯**：诞生的背景、技术/理念/需求来源、创始团队或核心推动者、当时的行业环境
2. **诞生节点**：明确的首次发布/成立时间，以及最初的形态和定位
3. **演进历程**：按时间顺序梳理所有关键节点（重大版本更新、融资事件、团队变动、战略转型、技术架构变化、用户规模里程碑、重大合作或收购、公关危机或争议事件）
4. **决策逻辑**：每个关键节点上，还原决策背后的原因（为什么选A而不是B？当时的约束条件是什么？）
5. **叙事要求**：不要写成干巴巴的年表。用故事的方式串起来，让读者感受到因果关系和时代脉络。

### ⚠️ Step 4: 横向分析（空间广度）

以当前时间点为切面，进行竞品对比分析。

**首先判断竞品情况**：

| 场景 | 条件 | 处理方式 |
|------|------|---------|
| 场景A | 无直接竞品（全新品类或独占性强） | 分析：为什么没有竞品？未来最可能从哪个方向冒出竞争者？有没有间接替代方案？ |
| 场景B | 少量竞品（1-2个） | 逐一深入对比，每个竞品展开详细分析 |
| 场景C | 竞品充分（3个及以上） | 选取3-5个最具代表性的进行对比，其余简要提及 |

**对比维度**（根据研究对象类型灵活调整）：
1. 核心差异对比：技术路线/核心方法论/底层逻辑、产品形态/商业模式/组织结构、目标用户/受众/适用场景、核心优势与明显短板、定价策略/资源投入/规模体量
2. 用户视角：真实用户口碑、社区评价、使用体验中被提及最多的优点和槽点
3. 生态位分析：在整个赛道版图中占据什么位置？填补了什么空白，还是在跟谁正面竞争？
4. 趋势判断：竞争格局中的走向、机会与风险

### ⚠️ Step 5: 横纵交汇总结

在报告最后，加一段「横纵交汇」的总结：
- 把纵向发展脉络和横向竞争格局结合起来
- 给出研究对象当前所处位置和未来走向的判断
- 不要写成前面内容的缩写版，要给出新的、综合性的判断

### Step 6: 输出格式

```
# [研究对象] 深度研究报告

## 一、纵向分析（发展史叙事）
### 1.1 起源与诞生背景
### 1.2 关键演进节点
### 1.3 决策逻辑分析

## 二、横向分析（竞品对比）
### 2.1 竞品格局判断
### 2.2 核心差异对比
### 2.3 用户视角分析
### 2.4 生态位与趋势

## 三、横纵交汇总结
### 3.1 当前所处位置
### 3.2 未来走向判断
```

## 写作风格要求

1. **可读性优先**：写得像优质深度报道或非虚构特稿，有节奏感，有画面感
2. **叙事驱动，不是罗列驱动**：有起承转合，有因果关系，不要流水账
3. **观点有据**：每个判断必须有事实支撑；推测需标注「（推测）」「（未经证实）」
4. **用人话写**：禁止"赋能""抓手""打造闭环"等空洞词汇，用具体细节代替概括性陈述
5. **对比有温度**：讲清楚每个竞品"活成了什么样"，用户选它的真实理由是什么

## 防 Anti-Patterns

- ❌ 不要写成"2023年1月发布了A，2023年3月发布了B"的流水账
- ❌ 不要写成咨询公司式的冷冰冰报告
- ❌ 不要用空洞形容词（如"赋能""抓手""打造闭环"）
- ❌ 不要罗列功能参数对比表，要讲"活法"不是讲"参数"
- ❌ 不要写未经证实的信息；如果某些信息无法确认，明确标注为推测
- ❌ 不要在结论部分重复前面的内容，要给出新的综合判断

## 篇幅参考

- **纵向分析**：6000-15000字（历史越长越靠近上限）
- **横向分析**：3000-10000字（竞品越多越靠近上限）
- **横纵交汇**：1500-3000字
- **全文总计**：10000-30000字
